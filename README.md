# Task 2 – Exploratory Data Analysis & Dashboard

## 📌 Objective
The goal of Task 2 is to perform exploratory data analysis (EDA) on a sales dataset, apply SQL queries to extract business insights, calculate key performance indicators (KPIs), and present findings in a static dashboard format.

## 🛠️ Steps Completed

### 1. Data Cleaning
- Standardized Gender values (Male/Female)
- Filled missing Age with median
- Replaced missing DOB with "Unknown"
- Removed duplicate records
- Converted PurchaseDate to datetime format

### 2. Descriptive Statistics & Univariate Analysis
- Generated summary statistics using `df.describe()`
- Plotted histograms for Age distribution
- Bar charts for Product frequency and Gender distribution

### 3. Revenue Calculation
- Added new column: `Revenue = Quantity × Price`

### 4. SQL Queries
- Top 5 Products by Revenue (overall)
- Top 5 Products by Revenue (last 6 months)
- Monthly User Acquisition Trend

### 5. Visualizations
- Revenue by Product (bar chart)
- Age Distribution (histogram)
- Revenue by Gender (bar chart)
- Monthly User Acquisition Trend (bar chart)

### 6. Multivariate & Correlation Analysis
- Scatter plots (Age vs Revenue, Quantity vs Revenue)
- Correlation heatmap
- Pair plot for variable interactions

### 7. KPI Calculations
- **Total Revenue:** 1,041,600  
- **Average Age of Customers:** 29.24 years  
- **Top Product (Last 6 Months):** Laptop  
- **Monthly New Users (Latest Month):** 18  

### 8. Dashboard Mock‑up
- One PowerPoint slide combining KPIs and charts into a static dashboard.

## ✅ Conclusion
Task 2 successfully demonstrates the complete workflow of data cleaning, exploratory analysis, SQL‑based business insights, and KPI reporting. The static dashboard consolidates findings into a clear, professional format that highlights customer demographics, product performance, and revenue trends. This task strengthens both technical and presentation skills, bridging academic learning with practical business analytics.
